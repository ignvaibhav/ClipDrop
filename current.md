# Current State of Island & Ferry

This document is the current working snapshot of the **Island desktop app** and **Ferry browser extension** as they exist in the codebase right now.

Updated: **May 14, 2026**

## 1. Product Identity

* **Desktop app:** `Island`
* **Browser extension:** `Ferry`
* **Main architecture:** Ferry injects download controls into YouTube and talks to Island over a local-only API on `127.0.0.1:49152`

## 2. Island Desktop App

Island is a Tauri + Rust desktop companion that exposes a local Axum API and WebSocket bridge.

### 2.1 Core API

Current endpoints in `app/src-tauri/src/server.rs`:

* `GET /health`
* `POST /formats`
* `POST /download`
* `POST /reveal`
* `GET /status/:job_id`
* `POST /action/open-settings`
* `POST /action/open-downloads`
* `GET/POST /ws`

### 2.2 Format Discovery

Format discovery is handled in `app/src-tauri/src/formats.rs`.

Current behavior:

* Uses `yt-dlp --dump-json`
* Builds three media groups:
  * `video`
  * `audio`
  * `thumbnail`
* Supports **Best** defaults for all three groups
* Includes richer metadata:
  * `format`
  * `quality`
  * `format_id`
  * `height`
  * `width`
  * `filesize`
  * descriptive `note`

Current important rules:

* Video options include labels like `Best • MP4 • 2160p • 4K • size`
* Audio options include labels like `Best • MP3 • 144 kbps • size`
* Thumbnail options include labels like `Best • JPG • 1920×1080`
* Multiple thumbnail qualities are now returned instead of collapsing to only one best thumbnail

### 2.3 Download Execution

Download execution is handled in `app/src-tauri/src/downloader.rs`.

Current behavior:

* Video and audio downloads still use `yt-dlp`
* Thumbnail downloads now support **direct selected thumbnail URL download**
* If a thumbnail option carries a direct URL in `format_id`, Island downloads that exact thumbnail instead of asking `yt-dlp` for a generic best thumbnail
* Filename variants are still generated by media type and quality

### 2.4 Native Actions

Current native integrations:

* `open settings`
* `open downloads folder`
* `reveal specific file/folder`

macOS reveal still uses:

* `open -R <path>`

## 3. Ferry Injected YouTube UI

The injected Ferry UI lives in:

* `extension/content.js`
* `extension/content.css`

### 3.1 Ferry Button

Current state:

* Ferry injects a native-looking button into the YouTube action row
* The button uses an inline SVG icon, not the extension PNG icon
* Button animation is horizontal:
  * open direction: left to right
  * close direction: right to left
* The icon stays flipped after closing until the next open action

### 3.2 Ferry Panel Model

The Ferry panel is currently implemented as a **true injected dropdown**, not a body-level popup overlay.

Current structure:

* Ferry button and panel live under a wrapper anchor in the YouTube action row
* The panel is appended inside the Ferry wrapper
* The wrapper uses `position: relative`
* The panel uses `position: absolute`

Current goal of this model:

* the panel should behave like a dropdown under the Ferry button
* the panel should follow the button while the page scrolls

There has been a lot of iteration here, so this area remains one of the more delicate pieces of the extension.

### 3.3 Ferry Panel Layout

Current header structure:

* Ferry logo
* Ferry text
* close button
* mode tabs below header

Current tabs:

* `video`
* `audio`
* `thumbnail`

The tab strip is icon-first and styled to feel more native to YouTube.

### 3.4 Video Tab

Current video tab behavior:

* video quality uses a `<select>` dropdown
* includes a `Best` option by default
* includes richer labels with:
  * container
  * resolution
  * quality badge
  * size
* supports clip trimming with:
  * slider
  * `From`
  * `To`
* has an explicit `Download video` button

Current quality badge mapping in Ferry:

* `4K` for `2160p+`
* `2K` for `1440p+`
* `Full HD` for `1080p+`
* `HD` for `720p+`
* `SD` for `480p+`
* `Low` for `360p+`
* `Very Low` below that

### 3.5 Audio Tab

Current audio tab behavior:

* audio bitrate uses a `<select>` dropdown
* includes a `Best` option by default
* supports clip trimming with its own slider state
* has an explicit `Download audio` button

Known fixed issues in this area:

* audio clip slider now initializes correctly when the tab opens
* audio end thumb / `To` time bug was fixed

### 3.6 Thumbnail Tab

Current thumbnail tab behavior:

* thumbnail preview is fetched **directly from YouTube**
* preview uses video ID and no longer waits for `yt-dlp`
* preview tries:
  * `maxresdefault.jpg`
  * fallback `hqdefault.jpg`
* thumbnail quality uses a `<select>` dropdown
* includes a `Best` option by default
* supports multiple thumbnail qualities
* has an explicit `Download thumbnail` button

## 4. Ferry Popup Window

The extension popup lives in:

* `extension/popup.html`
* `extension/popup.css`
* `extension/popup.js`

### 4.1 Current Popup Theme

Current popup design direction:

* monochrome black-and-white UI
* no gradients
* icon-led controls
* sharper inner card geometry
* dark root background to avoid white overscroll bleed

Current popup characteristics:

* solid black base
* white text/icons
* dark translucent white surfaces for cards and controls
* icon-based summary tiles
* icon-only clear button
* icon-led footer buttons

### 4.2 Scroll Behavior

The popup scroll issue was recently fixed.

Current behavior:

* `html` and `body` have fixed popup height
* `body` is non-scrolling
* `.activity` is the only scrollable region
* root overscroll is disabled
* activity overscroll is contained

Purpose:

* prevent white top/bottom bounce flashes
* keep the popup scrollable

### 4.3 Job Cards

Current job cards:

* show media type using icon-only badges
* show state using icon-only state chips
* use a progress bar
* show metadata line
* show `Reveal` as an icon-only button when a finished path exists

Current media/state rendering:

* `video`, `audio`, `thumbnail` are icon-based
* `queued`, `live`, `done`, `error` are icon-based
* icons are currently forced into the monochrome popup theme

## 5. Ferry Background / Sync Model

The extension coordination layer lives primarily in:

* `extension/background.js`
* `extension/runtime.js`

### 5.1 Activity Storage

Current activity model:

* activity is stored in extension local storage under `ferryRecentActivity`
* writes are serialized through a Promise queue
* this avoids race conditions between `queued`, `progress`, `done`, and `error`

### 5.2 WebSocket Sync

Current behavior:

* Ferry background opens a WebSocket to Island
* incoming backend events are broadcast immediately to the popup/content script
* progress writes are throttled for storage efficiency

### 5.3 Popup Reliability

Current popup reliability model uses two paths:

* fast path:
  * `WS_EVENT`
* durable path:
  * storage-backed activity list

Additional current fix:

* background now broadcasts `ACTIVITY_UPDATED` after queued/done/error writes finish
* popup reloads activity on `ACTIVITY_UPDATED`

This was added because some jobs, especially audio, could visually stay stale in the popup even when the backend had finished correctly.

## 6. Reveal / Open Actions

Popup action calls live in:

* `extension/api.js`

Current behavior:

* `openSettings()`
* `openDownloads()`
* `revealPath(path)`

Current hardening:

* these popup-triggered POST requests now use `keepalive: true`

Reason:

* the popup can close as soon as Finder or a desktop action steals focus
* without `keepalive`, local POST requests could be aborted and show `TypeError: Failed to fetch`

## 7. Known Recently Fixed Bugs

These have been fixed in the current code:

* thumbnail preview blocked by `yt-dlp` timing
* explicit format selector causing unavailable format failures
* audio tab clip slider not initializing correctly
* audio `To` time / right thumb not appearing
* popup list renderer crashing because `badge` variable was accidentally removed
* popup completion state failing to reconcile reliably across media types
* popup reveal/open requests being fragile when the popup closed immediately

## 8. Areas Still Sensitive / Worth Watching

These areas are still relatively sensitive because they have had repeated UI and behavior iteration:

* YouTube injected dropdown positioning and clipping
* Ferry action-row integration on YouTube SPA navigation
* popup activity reconciliation during fast multi-download sessions
* icon/theme consistency between monochrome popup design and richer branded surfaces elsewhere

## 10. Short Current Summary

Right now the product is in this shape:

* Island is a local desktop companion that probes formats, executes downloads, and exposes native actions
* Ferry injects a real YouTube download UI with separate video/audio/thumbnail flows
* Best defaults now exist for video, audio, and thumbnail
* Thumbnail preview is direct and fast
* Popup is now a monochrome downloads manager with icon-led cards and stronger state syncing
* The main remaining fragility is not the backend pipeline, but the highly dynamic frontend behavior inside YouTube’s SPA environment
