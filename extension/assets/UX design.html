05
UX Audit & Fixes
Fix
Status counts (Live/Error/Done) lack visual hierarchy
Replace plain text row with coloured status pills. Counts should live inside the pill. Current design requires reading 3 separate labels — pills communicate state at a glance.
Fix
Popup has no visual feedback when app is offline
Add a full-width banner: "Ferry desktop app not running. Open Ferry →" with a launch deep-link. Currently shows nothing, leaving users confused.
Fix
Quality list has no file size at a glance
Every quality row must show file size inline (e.g. "515 MB"). Users need this to decide — 4K on a 10min video is often 2GB+. This is the #1 decision factor.
Improve
Progress bar is absent during Queued state
Show the track at 0% always. Seeing an empty bar is more honest than seeing nothing. When download starts, animate fill with yt-dlp's progress output.
Improve
Clip time fields always visible, wasting space
Move From/To into a third "Clip" tab. Most users never clip — don't show it by default. Tab approach keeps the popup compact and focused.
Improve
Download button label is generic
Change "Download video" to "Download · 1.2 GB" — show the selected file size in the button. Makes users feel informed before they commit.
Add
No done state / success feedback in popup
After download completes, replace the download button with: "✓ Saved — Show in Finder" (or Explorer on Windows). Clicking opens the folder. Critical for UX closure.
Add
No keyboard shortcuts
Power users expect: Escape to close, Enter to start download, 1-4 to select quality. Add tooltip on hover showing the shortcut. Small effort, big feel improvement.
Add
Thumbnail not loaded in transfer items
Extract video thumbnail URL from the page metadata and show it in the 52×36 slot. Makes the history list scannable without reading titles. Already available as og:image.
06
Motion Principles
Easingcubic-bezier(0.16, 1, 0.3, 1) — expo out
Duration fast120ms — state changes, hover
Duration base220ms — popup open, tab switch
Duration slow400ms — progress bar fill
Popup entrancetranslateY(6px) → 0 + opacity 0 → 1
Progress fillwidth transition, linear easing only
No bounce, no spring. This is a utility tool — motion should feel precise and fast, not playful. Every animation should confirm an action, never decorate.
